## DEVELOPMENT OF HUMAN RESOURCE MANAGEMENT SYSTEM (HRMS) AND PAYROLL SYSTEM

**Date:** December 21, 2025
**Document Version:** 1.0
**Project Reference:** UNRE-HRMS-2025

---

## 1. PROJECT BACKGROUND

The University of Natural Resources and Environment (UNRE) requires a comprehensive, integrated Human Resource Management System (HRMS) and Payroll System to streamline HR operations, improve efficiency, and ensure accurate payroll processing for all university staff.

The current manual and semi-automated processes have resulted in:
- Time-consuming administrative tasks
- Data inconsistencies and errors
- Difficulty in generating timely reports
- Inefficient leave and attendance tracking
- Complex payroll calculations requiring automation

This Terms of Reference outlines the requirements for the development of a modern, web-based HRMS and Payroll System tailored to UNRE's specific needs and PNG regulatory requirements.

---

## 2. PROJECT OBJECTIVES

### 2.1 Primary Objectives
1. Develop a comprehensive, integrated HRMS and Payroll System for UNRE
2. Automate HR processes including recruitment, onboarding, performance management, and offboarding
3. Implement accurate payroll processing compliant with PNG tax and superannuation regulations
4. Provide real-time reporting and analytics capabilities
5. Ensure data security and user access control

### 2.2 Specific Outcomes
- **Efficiency Improvement:** Reduce HR administrative workload by 60%
- **Accuracy:** Achieve 99.9% payroll calculation accuracy
- **Compliance:** Full compliance with PNG tax laws and labor regulations
- **Accessibility:** 24/7 system access for authorized users
- **Reporting:** Generate comprehensive HR reports within minutes

#### J. Talent Management

#### K. Employee Relations

#### M. Travel and Expenses
- Travel Request Management

#### O. Administration and Security
- User Management and Role-Based Access Control
- Audit Trail and Activity Logging

### 3.2 Technical Specifications

#### Platform Requirements
- **Type:** Web-based application
- **Accessibility:** Cross-browser compatible (Chrome, Firefox, Safari, Edge)
- **Responsive Design:** Desktop, tablet, and mobile compatible
- **Hosting:** Cloud-based (Supabase/AWS/Azure/Google Cloud)
- **Database:** PostgreSQL or equivalent enterprise-grade RDBMS

#### Technology Stack (Reference)
- **Frontend:** Next.js 15, React 19, TypeScript
- **UI Framework:** Tailwind CSS, shadcn/ui components
- **Backend:** Supabase or equivalent
- **Authentication:** Secure user authentication with role-based access
- **Security:** HTTPS, data encryption, secure password storage

#### Integration Requirements
- **Banking Systems:** Export files for BSP, Westpac, ANZ, Kina Bank
- **Email Notifications:** Automated email alerts for approvals, notifications
- **Document Management:** Upload and store employee documents securely
- **Data Import/Export:** Excel and CSV import/export capabilities

### 3.3 Functional Requirements

#### User Roles and Access Control
1. **Super Administrator:** Full system access
2. **HR Manager:** HR module access, payroll processing
3. **Department Manager:** Department staff management, approval workflows
4. **Employee:** Self-service portal access

#### Security Requirements
- Multi-factor authentication (optional)
- Password complexity enforcement
- Session timeout management
- Audit logging of all system activities
- Data backup and recovery mechanisms
- Role-based access control (RBAC)

#### Performance Requirements
- Page load time: < 3 seconds
- Concurrent users: Minimum 100 simultaneous users
- System uptime: 99.5% availability
- Data backup: Daily automated backups
